# ТЗ на редизайн UI лендинга Shnurok

## 1. Контекст и цель

Текущий лендинг Shnurok уже решает продуктовую задачу: пользователь проходит короткий flow и получает подборку кроссовок. Менять механику, шаги, данные и логику не нужно.

Главная проблема текущего UI: стиль выглядит слишком игровым и детским для аудитории, которая приходит выбирать и покупать кроссовки. Нужно сохранить характер бренда, но сделать интерфейс более взрослым, магазинным и доверительным.

Целевой стиль: чистый e-commerce интерфейс в духе основного сайта [shnurok-shipping.ru](https://shnurok-shipping.ru/) и приложенного бренд-файла `shnurok презентация.ai`.

Основной принцип: меньше neo-brutal, больше аккуратного retail UI.

## 2. Источники визуального стиля

### Основной сайт

Ориентир по ощущению:

- каталоговый, покупательский интерфейс;
- белая база;
- черный текст;
- карточки товаров с фото, ценами, брендами и действиями;
- аккуратные разделители и контролы;
- минимум декоративного шума;
- интерфейс должен помогать выбрать и купить, а не выглядеть как развлекательный квиз.

### Бренд-файл

Файл: `/Users/stanislav/Downloads/shnurok презентация.ai`

Ключевые элементы из бренд-файла:

- шрифт: Inter Regular и Inter Bold;
- цвета: Outsole, Suede, Mesh, Cement, Lace;
- логотип SHNUROK как отдельный графический знак;
- line-art иллюстрации: браузер-маскот, руки, кроссовок, иконки;
- lockup `SNEAKER ADDICTED`;
- плашка `ОРИГИНАЛ`;
- общий стиль: минимальная палитра, много белого пространства, черный контур, голубой акцент.

Важно: брендовые иллюстрации использовать дозированно. Они должны добавлять узнаваемость, но не возвращать интерфейсу детское настроение.

## 5. Дизайн-токены

### Цвета

Ввести брендовые CSS-переменные:

```css
--outsole: #0A0A0A;
--suede: #656565;
--mesh: #B0DDFF;
--cement: #DDDDDD;
--lace: #FFFFFF;
```

Назначение:

- `--outsole`: основной текст, основные кнопки, активные рамки, иконки;
- `--suede`: вторичный текст, подписи, inactive labels;
- `--mesh`: единственный акцент, selected state, progress fill, мягкие фоновые плашки;
- `--cement`: разделители, нейтральные рамки, track progress bar;
- `--lace`: фон страницы, карточки, поля.

### Совместимость с текущими токенами

Не удалять старые токены резко. Сначала замапить их на новую систему:

```css
--ink: var(--outsole);
--concrete: var(--lace);
--cream: var(--lace);
--card: var(--lace);
--flame: var(--outsole);
--cobalt: var(--mesh);
--volt: var(--mesh);
```

После этого пройтись по `Hero.tsx`, `Flow.tsx`, `ui.tsx` и убрать старые классы `bg-volt`, `bg-flame`, `bg-cobalt`, `text-flame`, `bg-cream`, заменив их на семантически правильные классы или новые utility.

### Типографика

Подключить Inter:

- Inter Regular 400 для основного текста;
- Inter Bold 700 для заголовков, кнопок, цен, важных labels.

В `src/routes/__root.tsx` заменить подключение Google Fonts:

- убрать `Unbounded`;
- убрать `JetBrains Mono`;
- добавить `Inter:wght@400;500;600;700`.

В `src/styles.css`:

```css
--font-display: "Inter", system-ui, sans-serif;
--font-mono: "Inter", system-ui, sans-serif;
```

Важно:

- не использовать моноширинный шрифт в UI flow;
- не использовать `font-black`;
- не использовать тотальный uppercase для заголовков и кнопок;
- `letter-spacing` держать нормальным, без агрессивного `tracking-[0.24em]`;
- логотип SHNUROK не набирать текстом, использовать SVG.

### Геометрия

Общие правила:

- карточки: `rounded-xl` или `rounded-2xl`;
- кнопки: `rounded-full`;
- поля/чипы: `rounded-full` или `rounded-xl`;
- рамки: 1px, максимум 2px для активного состояния;
- никаких толстых рамок 4-5px;
- никаких offset shadows;
- никаких поворотов `rotate-*`.

### Тени и движения

Удалить:

- `shadow-[8px_8px_0_var(--ink)]`;
- `shadow-[10px_10px_0_var(--ink)]`;
- `shadow-[14px_14px_0_var(--ink)]`;
- `shadow-[16px_16px_0_var(--ink)]`;
- hover-сдвиги `hover:translate-x-*`, `hover:translate-y-*`;
- `animate-wiggle`;
- bouncy easing.

Допустимо:

- `shadow-sm` или `shadow-md` только для мягкого hover/focus;
- `transition-colors`;
- `transition-opacity`;
- легкий `slide-up` без прыжка.

### Фон страницы

Убрать grid-overlay из body:

- удалить `background-image` с линейной сеткой;
- оставить чистый `--lace`;
- для отдельных секций можно использовать очень светлый `#F7F7F7`, но базовая палитра должна оставаться брендовой.

## 8. Экран Hero

Файл: `src/components/shnurok/Hero.tsx`

### Текущая проблема

Hero сейчас выглядит как яркий playful-плакат:

- крупный uppercase;
- толстые черные рамки;
- синий blob с поворотом;
- стикеры с поворотами;
- лаймовый и оранжевый акценты;
- тяжелые brutal shadows.

Это и создает ощущение "чуть детского".

### Цель

Hero должен выглядеть как вход в умный сервис внутри магазина кроссовок.

### Header

Заменить текстовый логотип:

```tsx
Шнурок<span className="text-flame">.</span>
```

на SVG:

```tsx
<img src="/brand/logo-shnurok.svg" alt="SHNUROK" />
```

Ограничения:

- высота логотипа 32-40px desktop;
- 28-32px mobile;
- не растягивать по ширине;
- не добавлять точку, если ее нет в логотипе.

Справа можно оставить один компактный служебный chip:

- `алгоритм подбора`;
- или `beta`;
- или `sneaker matching`;

Стиль chip: тонкая рамка, без анимации.

### Eyebrow

Заменить `сервис умного подбора` на один из вариантов:

- `SNEAKER ADDICTED`, если используем брендовый lockup;
- `умный подбор кроссовок`, если нужен более понятный текст.

Если используется `SNEAKER ADDICTED`, не делать его огромным. Это должен быть eyebrow, а не второй логотип.

### Заголовок

Сохранить смысл, но убрать плакатную агрессию.

Рекомендуемый текст:

```text
Подберем кроссовки под стиль, размер и задачу
```

Стиль:

- `Inter Bold`;
- не uppercase;
- `text-5xl` desktop максимум, без сверхкрупного `5.5rem`;
- `leading-tight`;
- цвет `--outsole`;
- ключевое слово можно подсветить `--mesh` фоном с `rounded-md`, без рамки и тени.

### Описание

Сделать текст более покупательским:

```text
Ответьте на несколько вопросов, выберите визуальный стиль и получите подборку моделей, которые подходят по размеру, бюджету и сценарию носки.
```

Стиль:

- `text-base` или `text-lg`;
- цвет `--suede`;
- line-height 1.5-1.6.

### CTA

Кнопка `Подобрать кроссовки`:

- primary button;
- черный фон;
- белый текст;
- без стрелки `→`, если она выглядит декоративно;
- можно оставить короткую стрелку только если она не ломает стиль.

### Правая визуальная зона

Сохранить фото кроссовка как главный визуальный объект.

Изменить:

- убрать два повернутых cobalt-блока;
- добавить один спокойный blob/панель цвета `--mesh`;
- фото без поворота;
- рамка фото 1px `--outsole` или без рамки;
- радиус 16-24px;
- без offset shadow.

Floating chips:

- оставить 2-3 штуки максимум;
- `EU 42`;
- `до 15 000 ₽`;
- `на каждый день`;
- убрать `белый`, если визуально перегружает;
- без поворотов;
- без тени;
- цветовая система: white / mesh / black.

Маскот:

- добавить маленький `mascot-browser.svg` как secondary decoration;
- не делать маскота крупнее фото;
- лучше расположить в углу визуального блока или рядом с chip;
- на mobile можно скрывать или уменьшать.

Если показывается preview товара:

- оформить как товарную карточку;
- фото, бренд, модель, цена;
- без brutal рамки.

## 17. Экран search/loading

Step: `search`

### Цель

Показать, что идет подбор, но не делать экран слишком игровым.

### Визуал

- чистый белый фон;
- progress/status блок;
- 3-4 строки статусов в стиле checklist;
- active status можно подсветить `--mesh`;
- карточки preview с opacity/blur допустимы, но без offset shadows.

### Текст

Сохранить смысл текущих статусов, но подать спокойнее:

- `Сверяем размер`;
- `Уточняем бюджет`;
- `Подбираем модели`;
- `Готовим выдачу`.

Не использовать:

- агрессивный uppercase;
- огромные animated cards;
- яркие flame/volt состояния.

## 18. Экран results

Step: `results`

### Цель

Самый важный экран с точки зрения продаж. Он должен максимально приблизиться к e-commerce карточкам основного сайта.

### Общий layout

- Заголовок: `Подборка готова`;
- subtitle: коротко объяснить, что модели подобраны по профилю;
- слева/сверху можно показать компактный summary профиля;
- результаты группировать, но без ярких badge-заливок.

### Группы результатов

Текущие группы:

- `Точное попадание`;
- `Чуть дороже, но стоит`;
- `Альтернатива`.

Оставить, но сделать badges спокойными:

- default badge: border `--cement`;
- best match: background `--mesh`;
- premium: black text + white background, без flame.

### Карточка товара

Каждая карточка должна напоминать товарную карточку:

- фото товара на белом/светлом фоне;
- бренд small label;
- модель bold;
- цена bold;
- размеры chips;
- короткое объяснение `почему подходит`;
- CTA `Смотреть` или `Выбрать`;
- secondary action можно сделать иконкой избранного, если уже есть паттерн.

Стиль:

- border 1px `--cement`;
- radius 16px;
- hover: border `--outsole` или subtle shadow;
- без huge shadows;
- без uppercase-трекинга;
- CTA black button или outline.

### Акценты

Mesh использовать только:

- badge best match;
- hover/selected;
- маленькие статусные элементы.

Не заливать большие участки результатов голубым без необходимости.

## 19. Экран empty

Step: `empty`

### Цель

Если точной подборки нет, экран должен быть полезным, а не выглядеть как ошибка.

### Содержание

Оставить текущие действия:

- поднять цену;
- добавить цвета;
- смягчить стиль;
- посмотреть ближайшие варианты.

### Визуал

- белая карточка с понятным сообщением;
- действия как аккуратные chips/buttons;
- primary CTA: `Показать ближайшие варианты`;
- secondary CTA: `Начать заново`;
- можно добавить маленький outline-mascot, но не крупный.

Не использовать:

- черный большой блок с лаймовым текстом;
- flame/volt;
- brutal shadow.

## 20. Mobile адаптация

Проверить ширину 375px.

Общие требования:

- текст не выходит за контейнер;
- кнопки не сжимаются до нечитаемости;
- footer не перекрывает контент;
- Hero не занимает весь экран без возможности понять следующий шаг;
- изображения не обрезаются так, что товар невозможно рассмотреть;
- карточки результатов читаются как вертикальный список;
- сетка размеров и цветов остается удобной для тапа.

Минимальные размеры tap targets:

- кнопки: 44px высота минимум;
- карточки выбора: 48px минимум;
- chips: 36px минимум.

## 21. Desktop адаптация

Проверить:

- 1280px;
- 1440px;
- широкие экраны.

Требования:

- max-width не должен быть чрезмерным;
- карточки не растягиваются до пустых полотен;
- Hero выглядит собранным;
- результаты не теряют товарную плотность.

## 22. Доступность

Обязательные требования:

- focus-visible состояния для кнопок и карточек;
- контраст текста не ниже WCAG AA;
- alt для изображений;
- SVG-логотип с корректным `alt`;
- интерактивные карточки должны быть keyboard-accessible;
- disabled-состояния должны быть визуально и программно понятны.

## 24. Acceptance criteria

Редизайн считается готовым, если:

- лендинг визуально ближе к основному сайту Shnurok;
- исчезло ощущение детского neo-brutal плаката;
- осталась узнаваемость бренда через логотип, Inter, Mesh, outline-иллюстрации;
- флоу работает без изменений логики;
- все старые яркие акценты `volt/flame/cobalt` визуально заменены;
- нет толстых черных рамок 4-5px;
- нет offset shadows;
- нет поворотов карточек и chips;
- товарные результаты выглядят как покупательская выдача;
- мобильная версия не ломается;
- `npm run lint` и `npm run build` проходят.
