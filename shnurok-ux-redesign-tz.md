# ТЗ: UX/UI редизайн флоу Shnurok (адаптив «один экран — одно решение»)

Дата: 2026-06-02. Основано на аудите живого сайта на брейкпоинтах 375 / 768 / 1440 с замерами `getBoundingClientRect` и `scrollHeight`.

## 0. Цель

Каждый шаг-решение квиза должен **читаемо помещаться в один экран** (≈100 dvh) без вертикального скролла на мобиле (375), планшете (768) и десктопе (1440). Убрать раздутые карточки и внутреннюю пустоту, перенести первичный CTA в правильную зону, перестроить шаг свайпа. Экран выдачи (`Results`) — единственный, где вертикальный скролл допустим.

Гайдлайны: WCAG 2.5.5 / Apple HIG 44pt / Material 48dp (тач-таргеты), законы Фиттса и Хика, thumb-zone (Hoober), закон Якоба (консистентность), 8-pt сетка, длина строки 45–75 знаков.

## 1. Исходные дефекты (замеры overflow = насколько контент длиннее экрана)

| Экран | 375 | 768 | 1440 | Главная проблема |
|---|---|---|---|---|
| Hero | 0 | 0 | 0 | ок (эталон) |
| Размер | +35 | 0 | 0 | карточки 190×128 под 2 цифры; disabled-CTA над заголовком |
| Цвет | **+524** | +146 | 0 | 13 карточек 190×143; сабтекст «цвет»; |
| Бюджет | +191 | 0 | 0 | карточки 324×247 под одну строку |
| Задача | +461 | +86 | 0 | карточки 351×450; видна 1 из 2 |
| Спорт | **+664** | 0 | 0 | 7 карточек в 1 колонку по ~250px |
| Стиль (свайп) | +251 | — | +116 | свайп-карточка похоронена под текстом; на десктопе стартует с центра экрана |
| Empty | +20 | — | — | 6 крупных кнопок, ведущих в одно место; нет шапки |
| Results | +1516 | — | 0(1шт) | сетка 4 кол при 1–5 товарах зияет; «Нашли 1 моделей» |

Сквозные причины: (1) раздутые карточки, (2) первичный CTA «Далее» над контентом на мобиле/планшете и в правом верхнем углу на десктопе, (3) `flex items-center` плодит дыры на десктопе, (4) полезный aside только на `xl` (≥1280).

## 2. Требования к компоновке (StepShell)

1. Контейнер шага — **flex-колонка высотой 100 dvh**: `header` (лого+прогресс+чипсы+ссылка «Назад») → блок заголовка (компактный) → `content` (flex-1, min-h-0) → `footer` с первичным CTA.
2. **Первичный CTA «Далее»**:
   - мобайл/планшет — закреплённый нижний бар (sticky bottom, с safe-area), кнопка на всю ширину, высота 48–56px;
   - десктоп — в нижнем баре справа, ширина по контенту.
   - Никаких disabled-CTA над контентом.
3. **«Назад»** — компактная текстовая ссылка в шапке (рядом с прогрессом), не большой элемент.
4. Убрать вертикальное центрирование, дающее дыры: заголовок компактный (мобайл ≤ `text-3xl`), контент прижат к верху своей зоны с разумным отступом.
5. Полезный aside (таблица размеров / палитра / превью бюджета) — показывать с `lg` (≥1024). На `< lg` либо прятать, либо схлопывать в раскрывашку рядом с заголовком (минимально — прятать).

## 3. Требования по экранам

### Размер
- Сетка-чипы: высота карточки 56–72px. Колонок: 5 (мобайл) / 5–6 (планшет/десктоп). Все 10 размеров без скролла.
- Таблицу «см» — в раскрывашку/aside (lg+), не в высокую колонку только на xl.

### Цвет
- Swatch-грид: кружок 44–48px + подпись под ним. Колонок: 4 (мобайл) / 5 (планшет) / 6 (десктоп). 13 цветов + «Без разницы» в один экран.
- **Удалить сабтекст «цвет»** под каждой карточкой. Выбор — обводка + галочка, без смены всей карточки.

### Бюджет
- 2 колонки (мобайл) / 3 (десктоп). Высота карточки 72–88px. Превью «топ-пары» — строкой под сеткой или в aside (lg+), не колонкой только на xl.

### Задача (daily/sport)
- Обе опции видны сразу: 2 колонки на всех ширинах ИЛИ невысокие карточки 1-в-ряд с фото-превью 88–96px слева. Высота ≤ ~200px.
- Убрать дублирующую пилюлю-лейбл (заголовок и пилюля несут одно и то же).

### Спорт
- 2 колонки (мобайл), 3 (планшет), 4 (десктоп). Компактная карточка ~88–104px: иконка + название + саб, фото опционально ≤56px. 7 видов почти в один экран.

### Стиль (Tinder-свайп) — приоритет
- Убрать дубли: заголовок/подзаголовок экрана сжать в одну строку или убрать (прогресс-бар уже сообщает «стиль N из M»); **убрать пилюлю «стиль N из M»**.
- Название и описание стиля — **оверлеем на карточке** (низ, градиент), а не блоком над ней.
- Карточка занимает основную высоту вьюпорта; кнопки «Нравится/Не нравится» внизу без перекрытия карточки (нижний отступ колоды = высоте бара). «Назад» доступна и не конфликтует с баром.
- Свайп-логика, голоса и переход в search сохраняются без изменений.

### Поиск (loading)
- На мобиле — чек-лист сверху, превью-грид под ним (3-в-ряд мелко). Не ломать таймеры/переход.

### Результаты
- Адаптивные колонки: `repeat(auto-fill, minmax(240–260px, 1fr))` + ограничение ширины контейнера, чтобы 1–5 товаров не зияли пустыми колонками.
- **Починить склонение**: «N модель / модели / моделей».
- На мобиле допустимо 2 колонки для уплотнения.
- Сохранить: плоский список, чипсы фильтров, отображение выбранного цвета, отсутствие «% подходит/Почему подходит».

### Empty-state
- Вернуть шапку (логотип).
- Свести к честному набору действий: первичная «Показать ближайшие варианты» + «Подобрать заново» + ссылка на shnurok-shipping.ru. Не плодить кнопки, которые по факту делают одно и то же.

## 4. Критерии приёмки

- 375: размер, цвет, бюджет, задача, спорт, стиль → `scrollHeight − clientHeight ≤ 24px`.
- 768: те же → ≤ 0–24px.
- 1440: те же → 0; свайп-карточка целиком в экране; нет карточек выбора с >50% внутренней пустоты.
- Тач-таргеты кликабельных элементов ≥ 44px.
- `npm run build` и `npm run lint` зелёные.
- Сохранены: payload-логика, чипсы, флаг цены до шага цены, флоу переходов, отображение выбранного цвета.
- Тесты `responsive-layout.test.tsx` / `style-deck-ui.test.tsx` обновлены под новую компоновку (старые class-ассерты на `xl:absolute`, `pb-20`, `clamp(...)` заменяются на актуальные).
